Section I

Introduction

L'algèbre de Boole est la branche des mathématiques qui traite des opérations et fonctions sur des variables logiques. Elle fut inventée au XIXe siècle par le mathématicien britannique George Boole et est aujourd'hui au cœur de l'informatique et de l'électronique numérique.

Tout circuit numérique — d'un simple interrupteur à un microprocesseur moderne — repose sur les mêmes principes booléens. Comprendre cette algèbre, c'est comprendre comment les ordinateurs raisonnent, calculent, et prennent des décisions à partir de simples impulsions électriques.

Dans ce chapitre, nous allons construire progressivement les outils nécessaires : les opérations de base, les propriétés algébriques, la représentation par portes logiques, et les méthodes de simplification des fonctions booléennes.

Section II

Concepts fondamentaux

L'ensemble booléen B

En algèbre de Boole, on ne travaille qu'avec deux valeurs, appelées valeurs de vérité :

B = {0, 1} // FAUX = 0, VRAI = 1

Ces deux valeurs correspondent à des états physiques réels : absence ou présence de tension électrique, circuit ouvert ou fermé, etc.

Convention de notation

Les variables booléennes sont généralement notées avec des lettres minuscules (a, b, x, y…). Leur complément (inverse) est noté avec une barre au-dessus : ā. La conjonction s'écrit a·b ou ab et la disjonction a+b.

Priorité des opérateurs

Comme en arithmétique classique, les opérateurs booléens ont des priorités :

// Ordre de priorité (du plus fort au plus faible) : 1. NOT (complémentation) 2. AND (conjonction · ) 3. OR (disjonction + ) // Exemple : a + b·c ≡ a + (b·c) ≠ (a+b)·c

Remarque : NAND et NOR

La porte NAND est la plus simple à réaliser technologiquement. Fait remarquable : on peut construire toutes les fonctions logiques en utilisant uniquement des portes NAND (ou uniquement des portes NOR). Ce sont des portes dites « universelles ».

Section IV

Propriétés & Théorèmes

Ces propriétés permettent de manipuler et simplifier algébriquement les expressions booléennes. Elles se démontrent toutes par table de vérité.

Éléments neutres & nuls
  • x + 0 = x
  • x · 1 = x
  • x + 1 = 1
  • x · 0 = 0
Idempotence & Complémentarité
  • x + x = x
  • x · x = x
  • x + x̄ = 1
  • x · x̄ = 0
  • x̄̄ = x (double inversion)
Commutativité
  • x + y = y + x
  • x · y = y · x
  • x ⊕ y = y ⊕ x
Associativité
  • x+(y+z) = (x+y)+z
  • x·(y·z) = (x·y)·z
Distributivité
  • x·(y+z) = x·y + x·z
  • x+(y·z) = (x+y)·(x+z)
Absorption & Allongement
  • x·(x+y) = x
  • x+(x·y) = x
  • x+(x̄·y) = x+y
  • x·(x̄+y) = x·y

Théorèmes de De Morgan

Ces deux théorèmes sont fondamentaux pour transformer les expressions logiques, notamment pour passer d'une forme canonique à une autre :

x + y = x̄ · ȳ
Le NOR équivaut à AND des compléments
x · y = x̄ + ȳ
Le NAND équivaut à OR des compléments
Mémo De Morgan

Pour appliquer De Morgan : on inverse la sortie, on inverse chaque entrée, et on échange ET↔OU. C'est une règle qui s'applique à n'importe quel nombre de variables.

Section VI

Fonctions logiques & Circuits

Définition d'une fonction logique

Une fonction logique (ou fonction binaire) prend n variables booléennes en entrée et retourne une valeur booléenne :

f : Bn → B // Pour n variables d'entrée, il y a 2ⁿ combinaisons possibles. // Exemple : 3 variables → 2³ = 8 lignes dans la table de vérité

Deux façons d'exprimer une fonction

Toute fonction logique peut être décrite de deux manières équivalentes :

1. Table de vérité

Liste exhaustive : pour chaque combinaison des variables d'entrée, on donne la valeur de sortie. Utile pour définir ou vérifier une fonction.

2. Équation logique (expression algébrique)

On écrit la fonction sous forme d'une expression booléenne. Pour obtenir l'équation depuis la table, on prend chaque ligne où la sortie vaut 1 et on forme le produit des variables (complémentées si elles valent 0).

Section VIII

Tables de Karnaugh

Inventée par Maurice Karnaugh en 1954, la table de Karnaugh est une méthode graphique pour simplifier les expressions logiques. Elle fonctionne très bien jusqu'à 4–5 variables. Au-delà, on préfère d'autres méthodes (Quine-McCluskey).

Construction du tableau

Pour une fonction S à n variables, le tableau de Karnaugh a 2n cases. Les variables sont distribuées entre lignes et colonnes, codées en code Gray — un seul bit change entre cases adjacentes.

Pourquoi le code Gray ?

Le code Gray garantit que deux cases adjacentes dans le tableau ne diffèrent que d'un seul bit. Cela permet de former des groupements valides lors de la simplification. L'ordre des colonnes est donc toujours : 00 — 01 — 11 — 10 (et non 00-01-10-11).

Règles de simplification

1
Écrire en 1re forme canonique (Σ∏)

Identifier toutes les combinaisons où la sortie vaut 1 et écrire la somme des produits correspondants.

2
Remplir le tableau

Placer des 1 dans les cases correspondant aux mintermes, et des 0 dans les autres.

3
Former les groupements (boucles)

Regrouper les 1 adjacents en rectangles de taille 2k (1, 2, 4, 8…). Commencer par les plus grands groupes possibles. Chaque 1 doit appartenir à au moins un groupe. Un même 1 peut être dans plusieurs groupes (chevauchements autorisés). Le tableau est un tore : les bords gauche/droit et haut/bas sont adjacents.

4
Écrire l'expression simplifiée

Pour chaque groupe : éliminer les variables qui apparaissent avec leur complément dans le groupe. Faire la somme logique des résultats.

Règle de réduction des variables

Un groupement de 2k cases élimine k variables de l'expression. Pour 4 variables totales :
• Groupe de 1 case → 4 variables restantes
• Groupe de 2 cases → 3 variables restantes
• Groupe de 4 cases → 2 variables restantes
• Groupe de 8 cases → 1 variable restante
• Groupe de 16 cases → 0 variable (S = 1 constante)

Section IX

Exemples pratiques

Exemple 1 — Simplification à 3 variables

Exemple 1 · S(a,b,c) = Σ(2, 3, 6)

Fonction de départ :

S(a,b,c) = ā·b·c̄ + ā·b·c + a·b·c̄

Table de Karnaugh :

c\ab 00011110
0 0 1 1 0
1 0 1 0 0
Groupe rouge : ā·b + a·b = b (c varie)
Groupe vert : ā·b·c̄ (case isolée)

Le groupe rouge regroupe les cases 01 et 11 de la ligne c=0. Dans ce groupe, a prend les valeurs 0 et 1 → a disparaît. Reste : b·c̄.

La case verte (minterm 3) est couverte par le groupe rouge et peut être groupée avec m₃ formant le groupe {m₁, m₃} = ā·b. Après analyse :

S(a,b,c) = b·c̄ + ā·b // Simplification : S = b·(c̄ + ā) — encore plus compacte
Résultat

On passe de 3 produits de 3 variables à 2 produits de 2 variables. Le circuit passe de 6 portes AND + 2 portes OR à seulement 2 portes AND + 1 porte OR.

Exemple 2 — Simplification à 4 variables

Exemple 2 · S(a,b,c,d) = Σ(0,2,3,6,7,8,9,10,11,14,15)

Table de Karnaugh (16 cases) :

cd\ab 00011110
00 1 0 0 1
01 0 0 0 1
11 1 1 1 1
10 1 1 1 1
Groupe rouge (8 cases, ligne cd=11) → 1 variable : c
Groupe vert (8 cases, ligne cd=10) → 1 variable : c·d̄… → c
Groupe or (3 cases col 00+10, ligne cd=00 et 01) → ā·b̄ + b·c·d

Après analyse des groupements :

S(a,b,c,d) = c + ā·b̄ + b·c̄·d // Groupe 8 cases : c varie pas → c // Groupe 4 cases : c,d varient → a·b // Groupe 2 cases : a varie → b·c̄·d
Bilan de simplification

On passe de 11 minterm de 4 variables à une expression de 3 termes contenant au plus 3 variables. Gain considérable en nombre de composants.

Quiz de vérification

Testez vos connaissances
Quelle est la valeur de 1 · 0 + 1 en algèbre de Boole ? ▶
1 · 0 + 1 = 0 + 1 = 1
On applique d'abord la conjonction (AND prioritaire) : 1·0 = 0, puis la disjonction : 0+1 = 1.
Simplifiez x + x̄·y par le théorème d'allongement. ▶
D'après le théorème d'allongement : x + (x̄·y) = x + y.
Vérification : si x=0, y=1 → 0+1=1 et 0+1=1 ✓. Si x=1 → les deux valent 1 ✓.
Appliquez De Morgan à a·b + c. ▶
a·b + c = (a·b̄ + c̄)
De Morgan sur la somme : P+Q = P̄·Q̄. Ici P = a·b, Q = c. Donc : (a·b)̄ · c̄ = (ā+b̄) · c̄.
Dans un tableau de Karnaugh à 4 variables, un groupement de 4 cases laisse combien de variables dans le terme ? ▶
Un groupe de 4 cases = 22, donc k=2 variables sont éliminées. Il reste 4 - 2 = 2 variables dans le terme correspondant.
Pourquoi le tableau de Karnaugh est-il codé en code Gray et non en binaire naturel ? ▶
Le code Gray garantit qu'entre deux cases adjacentes dans le tableau, un seul bit change. Cela assure que des cases adjacentes correspondent bien à des mintermes qui ne diffèrent que d'une variable — condition nécessaire pour que les regroupements donnent des simplifications correctes.
